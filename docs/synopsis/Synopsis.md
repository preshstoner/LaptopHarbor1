eProject Synopsis – LaptopHarbor
Problem Statement
Finding and purchasing laptops online can be overwhelming due to scattered information, limited comparison tools, and inconsistent user experiences.
Customers often struggle to evaluate specifications, manage carts, and track orders efficiently.

Objective
LaptopHarbor aims to provide a streamlined e‑commerce platform for laptops, offering intuitive browsing, comparison, and purchasing features.
The goal is to simplify the buying process while ensuring transparency, reliability, and user satisfaction.

Technologies Used
Frontend: Flutter (cross‑platform mobile and web)
Backend: Node.js / Express (API services)
Database: MySQL (structured storage, ERD‑based design)
Version Control: Git & GitHub (Codespaces integration)
Documentation: Markdown, Draw.io (DFDs, ERD, Flowcharts)
Expected Output
A functional prototype of LaptopHarbor with modules for authentication, product management, cart, order processing, wishlist, and reviews.
Complete design documentation (DFDs, ERD, flowcharts, table structures).
A stable, user‑friendly application that demonstrates professional project workflow and collaboration.
